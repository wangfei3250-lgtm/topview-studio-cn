# 爆款工厂整合说明

把「爆款工厂 · AI 电商带货视频」的**真实后端**整合进本仓库（topview-studio-cn）。
本仓库原有的前端原型保持不动；爆款工厂作为一个**同进程挂载**的引擎接入，补齐原型缺的「能真正跑通的链路」。

## 它补齐了什么

原型的 `server.js` 是内存态 mock（重启清空）。爆款工厂带来可落地的真实能力：

- **商品 → 广告脚本**：商品页链接（公开 OG/JSON-LD）或手填卖点 → 钩子/痛点/卖点/促单 分镜脚本（大模型生成 / 本地规则兜底）。
- **出图 / 出片**：火山方舟 Seedream/Seedance、OpenAI gpt-image-1；**无 Key 时本地出商品卡 SVG + SMIL 占位片**，全流程零成本可演示。
- **多语言本地化**：字幕/口播翻译 + 分语言 SRT（内置 15 种语言）。
- **持久化**：`node:sqlite` 落本机（`factory/.data/`），不像原型重启即丢。
- **开放接入**：OpenAPI 3.1 + MCP（stdio / HTTP）+ 15 个 Agent 工具。

## 怎么跑

无需 `npm install`（零依赖，仅需 Node ≥ 22.5）：

```bash
npm start                 # 本仓库原服务（端口 4177），已自动挂载爆款工厂
# 打开工作台：
#   原型首页            http://localhost:4177/
#   爆款工厂工作台      http://localhost:4177/factory/
#   爆款工厂 OpenAPI    http://localhost:4177/api/factory/agent/v1/openapi.json

npm run factory           # 也可独立运行爆款工厂（端口 4499）
npm run factory:smoke     # 32 项冒烟测试（商品解析 + 全链路 + Agent + MCP）
```

## 挂载方式（命名空间隔离，零冲突）

`server.js` 仅两处受控改动：① 顶部动态 `import("./factory/server/mount.mjs")`；
② 请求入口对以下前缀分发给引擎（在原 `handleApi` 之前拦截）：

| 前缀 | 用途 |
|---|---|
| `/api/factory/*` | 工作台 REST（内部去掉 `/factory` 段后交给引擎原路由） |
| `/api/factory/agent/v1/*` | Agent API / MCP(HTTP) / OpenAPI |
| `/factory`、`/factory/*` | 爆款工厂工作台 H5 |
| `/factory-uploads/*` | 生成产物（图/视频/字幕） |

与本仓库现有的 `/api/projects`、`/api/agent/*`、`/api/tools/run` 等**互不冲突**。

## 把原型 UI 接到真实后端（建议下一步）

原型的这些入口可以改成调用爆款工厂接口（都在 `/api/factory` 命名空间下）：

- 首页模板「商品链接视频 / UGC 广告 / 批量带货」→ `POST /api/factory/commerce/product-to-video`
- 工具弹层「文生图 / 图生视频」→ `POST /api/factory/projects/:id/media`
- 短剧/工作台「配音字幕」→ `POST /api/factory/projects/:id/localize`、`GET /api/factory/projects/:id/srt?lang=`
- 工作台任务 → `GET /api/factory/task/:id`

完整工具清单见 `GET /api/factory/agent/v1/openapi.json`。

## 接入 Agent（MCP）

```bash
# stdio（先启动本仓库服务），指向挂载在客户服务上的命名空间：
TOPVIEW_BASE=http://127.0.0.1:4177 TOPVIEW_AGENT_BASE=/api/factory/agent/v1 \
  node factory/mcp/server.mjs --token <设置页 Agent Token>
```

HTTP MCP：`http://127.0.0.1:4177/api/factory/agent/v1/mcp`（Header `Authorization: Bearer <Token>`）。
