# 开源集成调研

本项目优先选择可商用风险较低、能服务短剧制作链路的开源项目。当前不直接复制大型项目源码，而是在产品内建立接入清单、任务入口和 API 预留点。

## 已加入产品内清单

| 项目 | 许可证 | 可用位置 | 当前处理 |
| --- | --- | --- | --- |
| Comfy API Simplified | MIT | ComfyUI 本机工作流、API Format workflow、队列提交 | 已接入开源增强中心与 Comfy 面板 |
| SortableJS | MIT | 分镜、任务、时间线拖拽排序 | 已加入增强任务，后续可安装或本地实现 |
| LiteGraph.js | MIT | 节点式工作流画布、导出 JSON | 已加入增强任务，适合替换当前简化画布 |
| ffmpeg.wasm | MIT | 浏览器端转码、抽帧、合并音频、导出预览 | 已加入增强任务，适合导出中心 |
| OpenReel Video | MIT | 多轨时间线、字幕、音频、转场、浏览器编辑器 | 已加入增强任务，作为时间线设计参考 |
| Transformers.js | Apache-2.0 | 本地素材打标签、字幕校对、摘要、分类 | 已加入增强任务，适合无 API 的辅助能力 |

## 接入原则

- API Key 仍只保存在本地后端，前端只显示是否配置。
- ComfyUI 必须粘贴 API Format workflow JSON，系统负责保存、排队和后续参数替换。
- 对大型开源项目先做适配层和任务入口，确认许可证、体积、依赖和运行方式后再引入源码或 npm 包。
- GPL 类项目只做外部工具调用或用户本机连接，不把源码并入本项目。

## 下一步建议

1. 用 SortableJS 或等价本地实现，给分镜和时间线加拖拽排序。
2. 用 LiteGraph.js 做可导出的节点图，替换当前静态 Comfy 节点。
3. 用 ffmpeg.wasm 做本地封面抽帧和轻量转码预览。
4. 参考 OpenReel Video 的多轨时间线结构，补字幕轨、音频轨、视频轨。
5. 用 Transformers.js 做素材自动标签和字幕风险检查。
