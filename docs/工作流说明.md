# 工作流说明

## 总体链路

```text
用户输入/上传素材
  -> 选择智能体或技能
  -> 生成角色与场景
  -> 生成剧情节奏
  -> 合成分镜图
  -> 配音字幕
  -> Seedance2.0 生成视频
  -> 工作台任务追踪
```

## 智能体分工

### Canvas

用于完整项目，从素材识别到角色、场景、分镜图和视频队列都由它统筹。

### V2

用于更快的短视频广告和社媒视频，重点是开场钩子、镜头效率和转化节奏。

### V1

用于轻量草案、快速起稿和简单结构测试。

## 模型配置

当前前端核心配置在 `app.js` 的 `agentCore` 中：

```text
imageModel: gpt-image-2
videoModel: Seedance2.0
```

## 本地服务职责

`server.js` 做两件事：

1. 提供静态页面服务
2. 提供轻量 API，方便前端保存任务和获取智能体计划

当前任务数据是内存态，重启服务后会清空。后续如果接真实后端，可以把 `/api/tasks` 替换成数据库存储。
